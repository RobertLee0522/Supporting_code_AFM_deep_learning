# 電子研究紀錄簿

## 標題
訓練資料集去卷積方向確認與圓柱孔最小半徑修正（1.5px → 3px）

## 日期
2026-06-16

## 研究目的 / 背景
使用者於 `train13`（5×5px 訓練探針版本）重新訓練後，檢視
`runs/train13/03_trap_dilation_check.png`（dilation 前後對照圖），發現
Input（dilated，模擬 AFM 掃描）影像中孔洞幾乎完全看不到（接近全平坦），
僅 Label（GT）可見小洞，因而提出疑慮：

> 「我覺得現在的問題是，我設定訓練資料的時候，輸入是大洞輸出的預測結果
> 應該是小洞才正確，我目前覺得是訓練上出問題，資料集的設定方向不對」

需確認：(1) 訓練資料 pair 的方向（X/y 何者該是大洞/小洞）是否設定錯誤；
(2) dilation 後孔洞幾乎不可見的現象是否代表訓練資料退化。

## 方法 / 過程

### 1. 物理方向推導
AFM 灰階膨脹（grey dilation）物理：

```
真實表面（孔洞大且深） ⊕ 探針形狀 → AFM 掃描影像（孔洞變小變淺）
```

探針有限曲率半徑無法完全探入窄/深孔洞，孔洞在影像中必然「變淺、變窄」。
故去卷積（deconvolution）的訓練 pair 方向應為：

```
X（輸入）= dilated 影像（模擬 AFM 掃描，孔洞小且淺）
y（標籤）= 真實表面（Ground Truth，孔洞大且深）
模型學習：X（小洞/淺洞）→ y（大洞/深洞）
```

驗證現有程式碼（`Supporting_code_AFM_deep_learning.py`）中
`X = dilated_*_stack`、`y = true_*_stack` 的賦值與 `grey_dilation()`
呼叫順序，確認程式碼方向與物理推導**一致**，排除「方向設定錯誤」的假設。

### 2. 孔洞幾乎不可見現象的根因分析
推導探針「有效填充半徑」公式（探針可完全填平孔洞的臨界條件）：

```
r_hole < sqrt(2 × R_tip × D) / px_nm
```

其中 `R_tip = 46.9 nm`（afm_gui.py 量得真實探針曲率半徑）、
`D` = 孔洞深度（nm）、`px_nm = 39.1 nm/px`。

代入訓練資料最大深度 `D = 125 nm`：

```
極限半徑 ≈ sqrt(2 × 46.9 × 125) / 39.1 ≈ 2.77 px
```

對照當時 `cylinder_hole_randomizer()` 小孔半徑設定範圍 `1.5–8.0 px`，
其中最小值 1.5px（59nm）**小於**填充極限 2.77px，代表這類最小孔洞在
grey dilation 後會被探針**完全填平**，dilated 影像趨近全平坦 —— 此即
dilation_check 圖中孔洞幾乎不可見的根因，而非方向設定錯誤。

### 3. 修正實作
- `cylinder_hole_randomizer()`：小孔半徑下限 `1.5px → 3px`（117nm，
  大於填充極限 2.77px），確保 dilated 訓練影像保有可見孔洞訊號。
- 於 `Supporting_code_AFM_deep_learning.py` 訓練開始時新增物理說明
  log 區塊，列印 grey dilation 方向與填充半徑公式計算範例，避免後續
  調整資料集設定時再次產生方向誤解。
- `config.txt` 輸出同步更新圓柱孔半徑說明為 `3–24 px`。
- `afm_gui.py`：重建 worker 結束（成功或失敗）後印出 `'='×52` 分隔線，
  避免多次重建結果在 log 區混在一起難以判讀。

## 結果 / 數據
重新訓練 `train14` 後，log 正確顯示：

```
Generating cylinder holes (r=3–24px=117–938nm 小+大混合, depth=60–145nm, ...)
```

`config.txt` 同步顯示：

```
圓柱孔半徑       : 3–24 px（小孔 70%: 3–8px≥117nm / 大孔 30%: 8–24px）
最小半徑說明     : 3px（117nm）> 探針填充極限2.77px@深度125nm，確保dilated有訊號
```

訓練過程資料正規化範圍：`X_train: [-0.180, 0.956]`、
`y_train: [0.086, 0.625]`，符合預期（背景 0nm → 0.625，孔洞深度對應
較低正規化值）。

## 結論 / 討論
1. 訓練資料 pair 方向**本身正確**（X=dilated 小洞 → y=GT 大洞），
   使用者疑慮為對 AFM 去卷積物理方向的誤解：去卷積是「還原孔洞」而非
   「縮小孔洞」。
2. dilation check 圖孔洞不可見的真正原因是**最小孔洞半徑小於探針填充
   極限**，使最小尺度孔洞在訓練集中趨近退化解（全平坦輸入）。調高最小
   半徑後可確保訓練訊號有效。
3. 此次修正同時補強了「自我說明」機制（訓練 log 解釋物理方向、GUI log
   分隔符），降低未來調參時誤判方向的風險。
